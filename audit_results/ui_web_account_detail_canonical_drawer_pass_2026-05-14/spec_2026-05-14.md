# UI_WEB_ACCOUNT_DETAIL_CANONICAL_DRAWER_PASS

Date: 2026-05-14
Branch: `codex/external-agent-lab-isolated`
Base before contour: `cc991b7`

## Goal

Add and stabilize a canonical right drawer for one account in the web UI. The drawer must explain lifecycle, pool, hold state, bounded checks, recent bounded timeline, available and disabled actions, danger action separation, and the latest command result without becoming a new runtime authority.

## Allowed Scope

- `wild_boar_proxy/web_design_ui/index.html`
- `wild_boar_proxy/web_design_ui/styles/overview.css`
- `wild_boar_proxy/web_design_ui/scripts/overview.js`
- `tests/test_web_design_ui.py`
- `audit_results/ui_web_account_detail_canonical_drawer_pass_2026-05-14/`

## Forbidden Scope

- `runtime.py`
- command adapter or allowlist changes
- live server changes
- desktop/native bridge work
- raw log reads
- raw state file reads
- auth file reads
- token or secret display
- arbitrary browser paths
- browser file picker
- new account lifecycle command semantics

## Behavioral Rules

- Opening the drawer is UI-only and must not dispatch a command.
- Row click and `Открыть детали` may open the drawer, but may not mutate runtime.
- Lifecycle action payloads must remain bounded to `ui_action + account_id`.
- Disabled actions must be visible with a reason and must not dispatch.
- `retire_account` must be visually isolated as danger and pass through confirmation.
- Last command result is shown as command-packet outcome only; it is not account truth until refresh.
- Timeline and counters are bounded packet or fixture summary only; no log tailing.

## Visual Acceptance

- Right drawer width: `<= 560px`.
- Drawer target bottom: `<= 1000px`.
- Drawer internal scroll: none for captured canonical states.
- No clipped button text.
- Visible drawer icons are Phosphor PNG images; no visible inline SVG icons.
- Confirmation modal fits the `1600x1000` frame.

## Captured States

- active account detail
- reserve account detail
- held account detail
- problem account detail
- missing account after refresh
- disabled action reasons
- last command success
- last command failed / refresh failed
- retire confirmation
