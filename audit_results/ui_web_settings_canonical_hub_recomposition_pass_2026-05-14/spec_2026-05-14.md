# Spec: UI Web Settings Canonical Hub Recomposition Pass

## Objective

Recompose the existing Settings screen from the closed `090975d` hub baseline into a final 3x3 product hub: nine compact readonly/deferred cards, including one `Data / Installer` card, with no config mutation, no path picker, no secret input, and no desktop/runtime changes.

## In Scope

- Settings screen markup inside `wild_boar_proxy/web_design_ui/index.html`.
- Settings hub CSS in `wild_boar_proxy/web_design_ui/styles/overview.css`.
- Settings subtitle/banner copy in `wild_boar_proxy/web_design_ui/scripts/overview.js`.
- Settings contract coverage in `tests/test_web_design_ui.py`.
- Baseline and after screenshots/metrics for Settings at `1600x1000`.

## Out of Scope

- Runtime behavior.
- Command adapter behavior.
- Allowlist changes.
- Live server changes.
- Desktop renderer or bridge work.
- Accounts, Diagnostics, Overview, Setup, import, and API route implementation work.
- New subpages or configuration editing.

## Constraints

- Settings remains `data-config-mode="readonly"` and `data-settings-subscreen-mode="hub-only"`.
- Settings contains no `data-ui-action`, live action class, account action class, onboarding action class, or API route action class.
- `Data / Installer` is one card only and remains deferred. It does not expose paths, file inputs, browser pickers, installer init commands, or saved/ready claims.
- Visible Settings icons are Phosphor PNG images, not inline SVG.
- The screen must fit `1600x1000` without scroll or clipping.

## Assumptions

- Existing common header/source pill behavior remains shared across product screens.
- Existing preview controls remain in the sidebar footer and are not part of the Settings main content.
- The current command packet model does not provide a Settings save/edit surface.

## Acceptance Criteria

- [x] Settings has exactly 9 hub cards.
- [x] Hub grid is 3 columns by 3 rows at `1600x1000`.
- [x] `Data / Installer` is present as a deferred card.
- [x] No active Settings command surface is present.
- [x] No config/path/secret input is present.
- [x] No visible inline SVG icons are present in Settings.
- [x] State matrix screenshots were captured for fixture healthy, degraded, down, stale, and live integration failure.
- [x] Metrics show `documentScrollHeight <= 1000`, `mainScrollHeight <= mainClientHeight + 1`, `settingsBottom <= 934`, `maxCardHeight <= 180`, and no detected overflow.

- tests: `node --check wild_boar_proxy/web_design_ui/scripts/overview.js`; `python3 -m unittest tests.test_web_design_ui -q`; `python3 -m unittest tests.test_web_design_ui tests.test_web_design_live_server tests.test_web_design_command_adapter -q`
- build: static web UI syntax and unit suites only; no package build required for this UI-only contour.
- manual: screenshot review of `audit_results/ui_web_settings_canonical_hub_recomposition_pass_2026-05-14/screenshots/after/fixture_healthy.png`.
- live evidence: live integration failure screenshot and metrics captured as readonly no-data/failure state.

## Open Questions

- Future Settings subflows remain separate contours and require admitted command surfaces before any mutation UI is exposed.
