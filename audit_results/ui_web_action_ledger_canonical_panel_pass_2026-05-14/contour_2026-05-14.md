## Goal

Make command results a single canonical UI surface: compact last action on working screens, detailed current-session ledger in a right panel, explicit refresh semantics, duplicate-submit guard, and no persistent/runtime-truth claims.

## Size / Risk

- Size: M
- Risk level: medium
- Mode: implementation
- Decision owner: owner canon

## Out Of Scope

- runtime engine
- command adapter

- desktop/native bridge
- redesigning all product screens

## Acceptance Criteria

- Ledger is opened as a right panel and marked current UI session only.
- Working screens show only compact last-action summary.
- `ok_refresh_pending` and `ok_refresh_failed` are not green.
- `ok_refresh_complete` is green only after canonical refresh is confirmed.
- Support artifact success may be blue, not runtime-health green.
- Duplicate in-flight submit creates a local `duplicate_blocked` entry and does not send a second fetch.
- Rendered ledger does not expose raw browser `command_id`, `argv`, full paths, secrets, raw logs, or raw JSON dumps.
- Screenshot matrix at `1600x1000` has no horizontal/vertical spill, no clipped controls, no visible inline SVG icons.

## Stop Conditions

- Ledger becomes persistent audit history.
- UI stores command results in local/session storage.
- UI displays raw argv/command_id/full paths/secrets.
- Refresh failure appears green.
- Duplicate guard still sends multiple command fetches.
- Runtime/adapter/allowlist files are touched.
