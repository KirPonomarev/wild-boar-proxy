# Spec: UI Web API Route Remove Disabled Canonical Flow

## Objective

Add the first bounded destructive route-registry flow in the web UI:
`disabled route -> readonly preflight summary -> strong confirmation -> api_route_remove -> action ledger -> canonical API routes refresh`.

The UI must not become a route config editor or the authority for disabled-state truth. Server-side preflight remains authoritative.

## In Scope

- Enable `api_route_remove` only for routes whose readonly packet proves `enabled === false`.
- Keep enabled, unknown, and unproven routes blocked with a visible reason.
- Add confirmation modal preflight summary for disabled-route removal.
- Keep browser action payload bounded to `ui_action` plus `route_id`.
- Treat command OK plus refresh pending/failed/mismatch as not green success.
- Add/update static and Node UI tests for gating, payload, confirmation, and refresh mismatch.
- Capture local visual/DOM evidence for API connections route-remove surfaces.

## Out of Scope

- API route create/update/builder.
- API Route Detail page/drawer.
- Raw route JSON/config editing.
- Token/secret entry or display.
- Browser path/file/stdin payloads.
- Command adapter or allowlist expansion.
- Runtime, desktop, native picker, or filesystem mutation work.

## Constraints

- UI may show only a readonly preflight summary.
- Server repeats disabled-state preflight before command dispatch.
- Success requires canonical API routes refresh and route absence.
- Route still present after refresh is `refresh_mismatch`, amber, not green.
- No raw command argv, command id, paths, secrets, raw JSON, provider/model mutation payloads, or config contents in the DOM.

## Assumptions

- Existing server metadata and command adapter already admit `api_route_remove`.
- Existing live server preflight already blocks enabled and unproven routes.
- Fixture healthy contains one enabled and one disabled route for web-preview state review.

## Acceptance Criteria

- [x] Disabled route row menu has a destructive remove entry.
- [x] Enabled route remove remains disabled/blocked.
- [x] Unproven/unknown route remove remains disabled/blocked.
- [x] Confirmation modal shows readonly preflight summary and server recheck copy.
- [x] Browser payload remains bounded to `ui_action` and `route_id`.
- [x] Refresh mismatch maps to amber `refresh_mismatch`.
- [x] Tests cover UI gating and ledger mismatch semantics.
- [x] No runtime, adapter, allowlist, desktop, or builder scope mixed in.
