# Spec: UI_WEB_DIAGNOSTICS_CANONICAL_RECOMPOSITION_PASS

## Objective

Recompose the web Diagnostics screen into a canonical operator detail screen: calm signal list on the left, selected signal details on the right, concise diagnosis, check chain, wide telemetry scale, latest records, and disabled/deferred actions. Preserve runtime and command boundaries.

## In Scope

- Diagnostics-only HTML structure.
- Diagnostics-only CSS layout, spacing, telemetry scale, and action presentation.
- Diagnostics copy for fixture/live/deferred states.
- UI tests that pin the Diagnostics command boundary and visual contract.
- Baseline and after screenshots plus DOM metrics at 1600x1000.

## Out of Scope

- Runtime behavior.
- Command adapter behavior.
- Allowlist changes.
- Live server changes.
- Desktop files.
- Accounts, Settings, and Overview changes beyond shared header order for Diagnostics visibility.
- Raw log parsing or direct state reads.

## Constraints

- The only admitted Diagnostics command remains `export_diagnostics`.
- Full check, lifecycle, journal, and danger actions remain disabled/deferred in this contour.
- Fixture telemetry is a bounded history view and must not claim runtime truth.
- Live integration failure must not reuse previous healthy fixture records or scale.
- Visible Diagnostics UI icons must remain image-based Phosphor assets.
- Main Diagnostics content must fit the 1600x1000 frame without vertical spill.

## Assumptions

- Web preview continues to expose fixture/live controls outside the main content plane.
- Export remains a support artifact action, not a runtime-health proof.
- Lifecycle actions are governed by existing Accounts command mapping and are not admitted from Diagnostics in this contour.

## Acceptance Criteria

- [x] Header uses `Обновить`, `Экспорт`, and quiet disabled `Полная проверка`.
- [x] Left signal panel is fixed at 368px and shows 5 stable signal rows.
- [x] Right detail is a vertical stack, not a two-column graph/records split.
- [x] Telemetry scale is full-width, uses 48 DOM/CSS ticks, and measures at least 680px.
- [x] Latest records sit below telemetry and do not overflow.
- [x] Action buttons are disabled/non-dispatching and do not clip text.
- [x] Support packet fields are no longer a visible main-viewport card.
- [x] Live failure shows deferred telemetry/records instead of fixture healthy history.
- [x] Diagnostics subtree has zero visible inline SVG icons.
- [x] `documentElement.scrollWidth <= 1600`, `documentElement.scrollHeight <= 1000`, `main.scrollHeight <= main.clientHeight + 1`, `diagnosticsScreen.bottom <= 934`.
