# UI_WEB_SETUP_CANONICAL_WIZARD_RECOMPOSITION_PASS Spec

## Goal

Recompose `Setup / Первичная настройка` into a safe first-run admission wizard. The screen shows readiness, admitted/deferred/blocked boundaries, and next-step reasons without performing setup.

## Scope

- Web design UI only.
- `setup` screen markup, styling, copy, and state copy.
- UI tests that lock the setup screen as inert and path-safe.
- Screenshot and metric evidence at `1600x1000`.

## Out Of Scope

- Runtime changes.
- Command adapter or allowlist changes.
- Filesystem scan.
- Browser-provided path input.
- File picker.
- Config write.
- Installer init.
- Desktop or native picker bridge.
- Deep `select-client` or `import-existing` implementation.

## Canon Decisions

- The wizard is an admission/readiness surface, not an installer.
- `Продолжить` and readiness checks stay disabled until a future setup proof packet and admitted command surface exist.
- `Закрыть` may be active because it is local UI dismissal only.
- Data directory text stays readonly and unconfirmed; no absolute/private paths are shown.
- Client selection is deferred until command-owned candidates exist.
- Live failure must not reuse fixture success as setup truth.

## Acceptance

- Setup has a 5-step rail: readiness, components, client, data, check.
- Setup section has no inputs, selects, file inputs, `data-ui-action`, or `live-action`.
- No active primary button exists on the setup surface.
- Only `Закрыть` is active in the setup surface.
- Visible setup icons use PNG assets; visible SVG count is zero.
- At `1600x1000`: document height <= 1000, main scroll height equals client height, setup bottom <= 934, overflows are empty.
- Screenshots exist for fixture healthy, degraded, down, stale, and live integration failure.

## Verification Inputs

- Baseline screenshots and metrics in `screenshots/baseline/` and `baseline_metrics.json`.
- After screenshots and metrics in `screenshots/after/` and `after_metrics.json`.
- Independent audit recorded in `independent_audit_2026-05-14.json`.
